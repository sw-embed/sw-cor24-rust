# COR24 Assembler (as24) Notes

## Overview

The `asld24` toolchain provides a COR24 assembler (`cor24-as`), linker (`cor24-ld`), and a
load-command compactor (`longlgo`). These are C programs built from source in
`docs/research/asld24/`. We do not have access to the COR24 C compiler; our goal is to
generate correct COR24 assembly from Rust.

## Building the tools

```bash
cd docs/research/asld24
make
```

This produces three binaries: `cor24-as`, `cor24-ld`, and `longlgo`.

## Usage

The assembler reads from **stdin** (not a filename argument).

### Flags

| Flag | Description |
|------|-------------|
| `-l` | Listing output (assemble only, no link) |
| `-S` | Assembly output |
| `-c` | Assemble only (object output) |
| `-O` | Optimize |
| `-a` | Additional option |

### Assemble with listing (no link)

```bash
./cor24-as -l < fib.s
./cor24-as -l < sieve.s
```

This prints addresses, hex opcodes, and the disassembled instructions. Useful for
verifying that our Rust-generated assembly encodes correctly.

### Assemble and link

```bash
./cor24-as < sieve.s
```

This runs the assembler followed by the linker, producing COR24 monitor load/go
commands on stdout. Only works for self-contained programs (no unresolved externals).

### Assemble, link, and compact

```bash
./cor24-as < sieve.s | ./longlgo
```

Consolidates load commands into longer lines for efficient transfer to the monitor.

## Example files

### fib.s

Fibonacci program compiled from C. Computes `fib(33)` and prints the result.

- **Links?** No -- references `_printf`, which requires a C library we don't have.
- **Assemble only:** `./cor24-as -l < fib.s`

Key patterns visible in the listing:

- Function prologue: `push fp` / `push r2` / `push r1` / `mov fp,sp` / `add sp,-N`
- Function epilogue: `mov sp,fp` / `pop r1` / `pop r2` / `pop fp` / `jmp (r1)`
- Function call: `push` args, `la r0,<func>`, `jal r1,(r0)`, `add sp,N`
- Recursive calls to `_fib` with `r2` as the preserved argument

### sieve.s

Sieve of Eratosthenes (1000 iterations, primes up to 8190). Compiled from C.

- **Links?** Yes -- self-contained with built-in I/O (`_putchr` talks directly to UART).
- **Assemble and link:** `./cor24-as < sieve.s | ./longlgo`
- **Assemble only:** `./cor24-as -l < sieve.s`

Notable details:

- `_putchr` does direct UART I/O at address `-65280` (0xFF0100), polling status register
- `_printn` does decimal conversion using a `_divdec` table (1000, 100, 10)
- `_putstr` walks a null-terminated string byte by byte
- `.comm _flags,8191` allocates the sieve array in BSS

## COR24 architecture quick reference

- **Address space:** 24-bit (16 MB)
- **GP registers:** r0, r1, r2 (only 3 truly general-purpose)
- **Special registers:** fp (r3), sp (r4), z (r5, always zero, compare-only), r6 (iv), r7 (ir)
- **Instruction sizes:** 1, 2, or 4 bytes (never 3)
- **Branch displacement:** PC-relative; PC is 2 ahead of the branch instruction at execute time
- **Stack:** grows downward; `push` decrements sp by 3 (word size), `pop` increments by 3
- **Word size:** 3 bytes (24 bits)
- **UART:** at 0xFF0100 (data) / 0xFF0101 (status)
- **Silent overflow:** wraps at 24 bits

## Using as24 to validate Rust output

To test assembly generated by our Rust pipeline:

1. Write the `.s` file with COR24 assembly syntax
2. Run `./cor24-as -l < yourfile.s` and inspect the listing
3. Compare hex opcodes against expected encodings
4. For self-contained programs, link with `./cor24-as < yourfile.s | ./longlgo` and
   load onto the FPGA via the monitor
