The assembler I gave you at first, did
not support "nop". You'll see that there is no nop defined yet in
all.txt. When you do "nop" in that assembler, it thinks "nop" is an
undefined symbol, that's why you get "00 00 00". I have since added
"nop", and the designated opcode for nop is 0xff (it's actually, in the
COR24 CPU, "zxt z,z"). That assembler is attached. "add r0,r0" is NOT a
no-op, it doubles what's in r0.

The features you say the assembler I sent you doesn't support, it does,
but you probably have some syntax issues. For example, you can't write
"0xff", but you can write "0ffh". "sxt" and "zxt" are definitely
supported, as is "sub sp,dddddd". I don't know what the problem is with
"la", you can see it is done all over the place with the code listings
I've given you.
